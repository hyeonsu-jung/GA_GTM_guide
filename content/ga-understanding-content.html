Google Analytics4 이해하기

GA4의 핵심 개념과 데이터 모델을 깊이 있게 이해합니다

GA4 데이터 모델

GA4는 이벤트 기반 데이터 모델을 사용하여 사용자의 모든 상호작용을 추적합니다.

이벤트 기반 측정

  • 이벤트: 사용자의 모든 행동 (페이지뷰, 클릭, 구매 등)
  • 매개변수: 이벤트에 대한 추가 정보
  • 사용자: 웹사이트를 방문하는 개인
  • 세션: 사용자의 연속된 활동 기간
💡 핵심 개념
GA4에서는 모든 것이 이벤트입니다. 페이지뷰, 클릭, 구매 등 모든 사용자 행동이 이벤트로 기록되며, 각 이벤트는 매개변수를 통해 상세 정보를 전달합니다.

측정 프로토콜

서버 사이드에서 GA4로 데이터를 전송하는 방법입니다.

Measurement Protocol

  • HTTP 요청: REST API를 통한 데이터 전송
  • 실시간 전송: 즉시 GA4에 데이터 반영
  • 오프라인 추적: 서버 로그, CRM 데이터 연동
  • 배치 전송: 대량 데이터 효율적 전송

API 활용

📊 Reporting API
GA4 데이터를 외부 시스템으로 가져와서 커스텀 대시보드나 보고서를 생성할 수 있습니다.

사용자 중심 측정

GA4는 사용자 중심의 측정 방식을 채택하여 더 정확한 사용자 여정을 추적합니다.

User-ID
로그인한 사용자의 고유 ID를 설정하여 여러 기기에서의 활동을 연결합니다.
Google Signals
Google 계정을 통한 크로스 디바이스 추적과 인구통계학적 데이터를 제공합니다.
Enhanced Measurement
자동으로 스크롤, 클릭, 동영상 시청 등의 이벤트를 추적합니다.

데이터 수집 원리

GA4가 어떻게 데이터를 수집하고 처리하는지 이해해보세요.

쿠키와 개인정보 보호

  • 쿠키 기반: 기본적인 사용자 식별
  • 쿠키 없는 측정: 개인정보 보호 강화
  • Consent Mode: 사용자 동의에 따른 데이터 수집
  • 데이터 보존: 설정 가능한 데이터 보존 기간

데이터 처리

🔄 실시간 처리
GA4는 실시간으로 데이터를 처리하여 즉시 보고서에서 확인할 수 있습니다. 또한 머신러닝을 통해 데이터 품질을 향상시킵니다.
