이벤트 설정

사용자 상호작용을 정확하게 추적하기 위한 이벤트 구성

GA4 이벤트 개요

GA4에서는 모든 사용자 행동이 이벤트로 추적됩니다. 기본 이벤트와 커스텀 이벤트를 설정하여 비즈니스 목표에 맞는 데이터를 수집할 수 있습니다.

이벤트 유형

  • 자동 수집 이벤트: GA4가 자동으로 추적하는 이벤트
  • 향상된 측정 이벤트: 설정으로 활성화하는 이벤트
  • 커스텀 이벤트: 직접 정의하는 이벤트
  • 권장 이벤트: Google이 권장하는 이벤트

클릭 이벤트

버튼, 링크, 이미지 등의 클릭을 추적하여 사용자 행동을 분석합니다.

🖱️ 버튼 클릭
CTA 버튼, 다운로드 버튼 등의 클릭을 추적하여 전환율을 측정합니다.
🔗 링크 클릭
외부 링크, 내부 링크 클릭을 추적하여 사용자 흐름을 파악합니다.
🖼️ 이미지 클릭
상품 이미지, 배너 이미지 클릭을 추적하여 관심도를 측정합니다.

구현 방법

GTM을 통한 클릭 추적
// GTM에서 클릭 트리거 설정
// CSS 선택자: .cta-button, .download-link
// 이벤트 이름: button_click, link_click

스크롤 이벤트

페이지 스크롤 깊이를 측정하여 콘텐츠 참여도를 분석합니다.

스크롤 추적 설정

  • 25% 스크롤: 페이지의 1/4 지점 도달
  • 50% 스크롤: 페이지의 절반 지점 도달
  • 75% 스크롤: 페이지의 3/4 지점 도달
  • 90% 스크롤: 페이지 하단 근처 도달
📊 활용 방법
스크롤 데이터를 통해 콘텐츠의 품질과 사용자 관심도를 측정하고, 페이지 레이아웃을 개선할 수 있습니다.

동영상 이벤트

YouTube, Vimeo 등 동영상 콘텐츠의 시청 패턴을 추적합니다.

▶️ 동영상 시작
동영상 재생 시작 시점을 추적하여 콘텐츠 관심도를 측정합니다.
⏸️ 일시정지
동영상 일시정지 패턴을 분석하여 사용자 행동을 파악합니다.
⏹️ 완료
동영상 시청 완료율을 측정하여 콘텐츠 품질을 평가합니다.

폼 이벤트

문의 폼, 회원가입 폼 등의 제출과 오류를 추적합니다.

폼 추적 이벤트

  • form_start: 폼 작성 시작
  • form_submit: 폼 제출 시도
  • form_success: 폼 제출 성공
  • form_error: 폼 제출 오류
📝 구현 예시
// 폼 제출 이벤트
gtag('event', 'form_submit', {
  'form_name': 'contact_form',
  'form_id': 'contact'
});

이벤트 디버깅

설정한 이벤트가 올바르게 작동하는지 확인하는 방법입니다.

GA4 실시간 보고서

  1. GA4 인터페이스에서 "실시간" 메뉴 클릭
  2. 이벤트 섹션에서 발생한 이벤트 확인
  3. 이벤트 매개변수와 값 검증

브라우저 개발자 도구

  1. F12로 개발자 도구 열기
  2. Network 탭에서 gtag 요청 확인
  3. Console에서 이벤트 전송 로그 확인
🔍 GTM 미리보기
GTM의 미리보기 모드를 사용하면 실시간으로 이벤트 발생을 확인하고 디버깅할 수 있습니다.
